# Security Policy

We take the security of our systems and community data seriously. This document outlines our policy regarding the reporting of security vulnerabilities.

## Supported Versions

Currently, we only support security updates for the latest version deployed from the `main` branch.

| Version              | Supported          |
| :------------------- | :----------------- |
| Main / Production    | :white_check_mark: |
| Development / Others | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this website, please **do not** open a public issue on GitHub. This is to prevent exploitation before we can apply a fix.

Please follow these steps:

1.  Send a detailed report via email to: **novenoandrewm@gmail.com**.
2.  Include steps to reproduce the vulnerability (Proof of Concept).
3.  We will review your report and respond within 48 hours.
4.  We will notify you once a patch has been successfully applied.

## Disclosure Policy

- Please do not publish or share the details of the vulnerability with third parties until we have released a fix.
- Attacks that damage data integrity, disrupt service (DDoS), or steal private user/congregation data are strictly prohibited.

Thank you for helping keep the Vihara Avalokitesvara Pondok Cabe digital community safe.
